## Context

书签 item 当前在悬浮/长按时右上角显示删除按钮（叉号，绝对定位）。编辑按钮需以相同方式在左上角呈现。点击后需要弹出一个表单，预填当前书签的分组、名称、链接，供用户修改后保存。

项目现有添加书签的 modal（`#bm-modal`），包含分组、标题、URL 三个字段，可复用其逻辑。

## Goals / Non-Goals

**Goals:**
- 书签悬浮/长按时左上角显示编辑按钮
- 点击编辑弹出 modal，预填当前书签数据
- 确认后更新 localStorage 并重新渲染书签列表

**Non-Goals:**
- 不支持批量编辑
- 不做拖拽排序
- 不修改书签 favicon（favicon 由 URL 自动决定）

## Decisions

### 编辑 UI 方式

**决定**：复用现有的 `#bm-modal` 弹窗（添加书签弹窗），切换为"编辑模式"——预填数据、确认按钮改为"保存"、保存时更新而非新增。

**理由**：避免引入第二个弹窗，结构一致，改动最小。

**方案对比**：
- 内联编辑（原地展开表单）：布局复杂，影响周围书签排列
- 独立编辑弹窗：代码冗余，与添加弹窗高度重复
- 复用添加弹窗 ✓：改动最小，视觉一致

### 编辑按钮定位

**决定**：与删除按钮（右上角）对称，编辑按钮绝对定位在书签 item 左上角，同样在悬浮/长按时显示。

**理由**：与现有删除按钮行为一致，无需额外交互模式。

## Risks / Trade-offs

- [低] modal 复用需要在 JS 中区分"新增"和"编辑"两种模式，增加少量状态管理复杂度 → 用一个 `editingId` 变量标记当前编辑的书签 ID，null 表示新增模式
