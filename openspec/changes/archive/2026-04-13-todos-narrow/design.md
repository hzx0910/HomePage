## Context

页面当前布局：搜索区在上，书签区和待办区并排在下（flex row）。待办区当前与书签区各占一定宽度比例。用户希望待办区宽度缩小一半，让更多空间留给书签。

## Goals / Non-Goals

**Goals:**
- 待办区域宽度缩小至约原来的一半

**Non-Goals:**
- 不调整书签区布局
- 不修改待办功能逻辑
- 不做响应式适配

## Decisions

### 宽度调整方式

**决定**：找到待办区域对应的 CSS flex/width 属性，将其缩小为原来的一半。具体值取决于现有代码，通常是调整 `flex` 比例或设置固定 `width`。

**理由**：改动最小，只需修改一个 CSS 属性值。

## Risks / Trade-offs

- [低] 宽度缩小后，若待办内容较多，可能出现文字截断或滚动条——属于可接受的视觉权衡
