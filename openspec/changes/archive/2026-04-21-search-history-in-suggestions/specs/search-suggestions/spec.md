## MODIFIED Requirements

### Requirement: 展示搜索联想词列表
系统 SHALL 在搜索框下方展示一个悬浮联想词列表。用户在搜索框输入内容后（防抖 300ms），系统向当前默认搜索引擎的 Suggestions API（JSONP）发起请求，并将返回的联想词渲染为可点击列表项。渲染时，系统 SHALL 先从搜索历史中筛选出**包含当前输入词**的条目（最多 3 条），将其以历史条目样式（`.history-item`，含删除按钮）插入列表最前面，再渲染远程联想词；历史条目与联想词合计最多展示 11 条（历史 ≤3 + 联想词 ≤8）。输入框为空时列表 SHALL 隐藏（展示历史视图由 search-history 能力负责）。

#### Scenario: 有匹配历史时历史条目前置
- **WHEN** JSONP 请求返回联想词，且搜索历史中存在包含当前输入词的条目
- **THEN** 匹配的历史条目（最多 3 条）以历史样式显示在联想词列表最前面，其后紧跟远程联想词

#### Scenario: 无匹配历史时仅显示联想词
- **WHEN** JSONP 请求返回联想词，且搜索历史中无包含当前输入词的条目
- **THEN** 联想词列表仅显示远程联想词，与原有行为一致

#### Scenario: 输入触发联想词请求
- **WHEN** 用户在搜索框输入非空内容，且停止输入超过 300ms
- **THEN** 系统向当前默认搜索引擎的 JSONP Suggestions API 发送请求，请求参数为当前输入内容

#### Scenario: 展示联想词
- **WHEN** JSONP 请求成功返回联想词数组
- **THEN** 搜索框正下方出现联想词列表，列表宽度与搜索框对齐，最多展示 8 条远程联想词（加历史共最多 11 条）

#### Scenario: 清空输入时展示历史
- **WHEN** 用户清空搜索框内容（存在搜索历史）
- **THEN** 联想词列表隐藏，下拉区域改为展示搜索历史列表

#### Scenario: 清空输入且无历史时隐藏列表
- **WHEN** 用户清空搜索框内容，且搜索历史为空
- **THEN** 下拉列表完全隐藏

#### Scenario: API 请求失败时静默处理
- **WHEN** JSONP 请求失败（网络错误、超时、API 不可用）
- **THEN** 联想词列表不显示，搜索框正常可用，无错误提示
