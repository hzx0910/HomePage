## Context

当前搜索区域是单输入框 + 单"搜索"按钮，引擎由快捷词或 `defaultEngine` 配置决定。本次改为固定展示三个引擎按钮（百度、Bing、Google），UI 更直观，Enter 键仍走默认引擎。

## Goals / Non-Goals

**Goals:**
- 搜索区域显示三个引擎按钮，点击各自触发对应引擎搜索
- 默认引擎按钮有视觉标记（高亮/加粗）
- 点击非默认引擎按钮时可一键将其设为默认（长按或右键区域，或按钮旁小图标）
- 默认引擎选择持久化到 localStorage（复用 `homepage_config.search.defaultEngine`）
- 保留 Enter 键使用默认引擎搜索

**Non-Goals:**
- 不允许用户增删引擎（固定三个）
- 不保留原单按钮 UI

## Decisions

### 三按钮布局

**决定**：搜索框右侧并排三个按钮，每个按钮显示引擎名称（百度 / Bing / Google）。

**理由**：直观、零学习成本。相比下拉选择器，三个并排按钮可直接点击目标引擎，操作步骤更少。

### 默认引擎标记与切换

**决定**：默认引擎按钮样式高亮（使用 `--accent` 色），非默认引擎为普通样式。点击非默认引擎按钮时：
1. 立即以该引擎执行本次搜索
2. 同时将其设为新的默认引擎并持久化

**理由**：把"搜索"和"设默认"合并为一个操作，减少交互步骤。用户点了哪个就说明偏好哪个。

**替代方案**：点击只搜索，另设切换按钮 → 操作更繁琐，否决。

### 配置持久化

**决定**：复用 `homepage_config.search.defaultEngine` 字段，点击按钮后立即调用 `Storage.save()`。

**理由**：无需新增数据字段，与现有配置结构一致。

### Enter 键行为

**决定**：Enter 键始终使用当前 `defaultEngine`，与三按钮状态联动（高亮按钮 = Enter 的目标引擎）。

## Risks / Trade-offs

- [三按钮在小屏幕可能换行] → 当前 Non-Goal（不做移动端适配），可接受
- [点击即改默认可能误触] → 用户刷新后默认引擎已变，但操作可逆（再点回去），可接受
